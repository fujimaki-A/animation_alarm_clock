# 省電力化対策まとめ

## 既に実施済みの対策

### 1. 超音波センサー測定間隔の最適化

- **変更内容**: アラーム非活性時のチェック間隔を5秒に拡大
- **実装**: `IDLE_CHECK_INTERVAL_MS = 5000` 定数追加、`loop()`関数で動的に切り替え
- **効果**: センサー電力約90%削減（10mA → 2mA）
- **該当行**: [91行目](code/Clock.ino:91), [235-254行目](code/Clock.ino:235)

### 2. OLED表示更新頻度の削減

- **変更内容**: 時刻が変わった時だけ更新
- **実装**: `lastDisplayedHour/Minute` 変数追加、`displayData()`関数で更新判定
- **効果**: 表示更新回数1/60に削減、OLED電力約30-50%削減
- **該当行**: [137-138行目](code/Clock.ino:137), [684-705行目](code/Clock.ino:684)

### 3. アニメーション間隔の延長

- **変更内容**: アニメーション間隔を500ms → 1000msに変更
- **実装**: `ANIMATION_INTERVAL_MS = 1000`、`SAKURA_ANIMATION_INTERVAL_MS = 1000`
- **効果**: アニメーション電力約50%削減
- **該当行**: [94-96行目](code/Clock.ino:94)

### 4. LED輝度削減

- **変更内容**: LED輝度を32 → 16に変更
- **実装**: `LED_BRIGHTNESS = 16`
- **効果**: LED電力50%削減（256mA → 128mA）
- **該当行**: [97行目](code/Clock.ino:97)

## 今後の追加対策（効果大）

### 5. WiFi接続戦略変更（推奨度高）

- **内容**: NTP同期後にWiFi切断、アラーム設定変更時のみ再接続
- **効果**: WiFi電力（80-150mA）をほぼ完全に削減
- **労力**: 中程度
- **注意点**: Webアクセス時は一時的に再接続が必要

### 6. Deep Sleepモード導入（効果極大）

- **内容**: Arduino UNO R4 WiFiのスリープ機能を活用
- **実装**: 1分ごとに awake → 時間確認 → sleep
- **効果**: 平均電力90%削減（動作時100mA → スリープ時数mA）
- **労力**: 高
- **注意点**: スリープ中は全機能停止、RTCで wake-up

### 7. OLEDコントラストの最適化

- **内容**: 日中もコントラストを低下（`OLED_CONTRAST_DAY = 0x40`）
- **効果**: OLED電力さらに20-30%削減
- **労力**: 低

### 8. 部分更新の活用

- **内容**: `display.display()`の代わりに部分更新モード使用
- **効果**: 表示更新時の電力削減
- **労力**: 中

## 優先順位（効果/労力比）

1. **WiFi接続戦略変更** - 高効果、中労力
2. **OLEDコントラスト最適化** - 中効果、低労力
3. **部分更新の活用** - 中効果、中労力
4. **Deep Sleep導入** - 极高効果、高労力（ただし大幅な設計変更が必要）
